Represents a live, transmutable view over an underlying unordered collection of serialized items with set semantics — no ordering, no duplicates. Each element is transformed via a codec between the storage representation (TStorage) and the exposed runtime representation (TExposed).

Remarks

This type acts as a projection layer between an underlying set-backed storage container and a higher-level runtime representation.

Unlike typical adapter collections, this set does not maintain its own backing storage. Instead, all operations delegate directly to externally supplied accessors. This ensures that the collection always reflects the current state of the underlying storage, even if modified externally (for example by other mods).

Each element is encoded and decoded on demand using the supplied codec delegates. No caching is performed by this class.

Implementations using hash-set-backed storage typically achieve the following complexity characteristics: 

<ul><li>Add: <code>O(1)</code></li><li>Remove: <code>O(1)</code></li><li>Contains: <code>O(1)</code></li><li>Clear: <code>O(n)</code></li></ul>
